Request and Response

リクエストとレスポンス

Webアプリ全般に当てはまる構成要素

• Webブラウザはサーバ（Webアプリが動くコンピュータ）にリクエストを送信
• サーバはリクエストに応じた処理をしてレスポンスを返却
• Webブラウザはレスポンスに含まれるHTMLを解釈して表示

ビュー (View)

URL設定とビューの動き

1. URL設定から呼び出される（リクエストの情報も渡る）
2. リクエストの情報を使った処理をする（送信されたデータを使うなど）
3. レスポンスを返す

Comments: 0

published: 2020年2月29日13:07

テンプレート (Template)

URL設定とビューの連携にテンプレートを追加

1. URL設定がパスに対応するビューを呼び出し
2. ビューが処理
3. ビューがテンプレート（HTML）を取得し、レスポンスとして返す（HTMLに処理結果を埋め込むことも可能）

モデル (Model)

URL設定・ビュー・テンプレートの連携にモデルを追加

1. URL設定がパスに対応するビューを呼び出し
2. ビューが処理（モデルを使ってデータベースからデータの取得や保存をする）
3. ビューがテンプレートを取得し、処理結果を埋め込んでレスポンスとして返す（データベースのデータが入る）
